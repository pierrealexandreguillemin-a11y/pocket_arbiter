# Pocket Arbiter — LLM Fine-Tuning & RAG

## LLM Fine-Tuning Findings (Pocket Arbiter, session 2026-03-23)

### Loss Curve Analysis — CRITICAL
**NEVER read loss curves step-by-step.** Per-step loss is noisy (variance batch, batch_size=1).
ALWAYS compute moving average (window 10+) before concluding overfit or selecting checkpoints.

**Error made:** Interpreted step 60 (loss 1.97) -> step 80 (loss 2.19) as "overfit remontee".
Moving average showed continuous descent (2.13->2.08). Checkpoint selected too early -> under-learning.

### SFT Budget on Small Models (270M)
| Config | Budget (steps x LR) | Result |
|--------|---------------------|--------|
| v2 step 60 | 60 x 1e-5 = 0.0006 | Under-learns (70% responses < 10 words) |
| v2 full | 102 x 1e-5 = 0.00102 | Never evaluated (cut too early) |
| **v3 step 140** | **140 x 1e-5 = 0.0014** | **Sweet spot (MA loss 1.716, acc 0.620, overfit 1.08)** |
| v3 full | 204 x 1e-5 = 0.00204 | MA plateau/slight rise after step 144 |
| v1 final | 306 x 2e-5 = 0.00612 | Over-learns (echo 17.6%, overfit ratio 1.33) |

### Checkpoint Selection Protocol
1. Compute moving average loss (window 10)
2. If MA still descending -> model still learning -> do NOT stop
3. Use save_steps for intra-epoch checkpoints (save_only_model=True for disk)
4. Select checkpoint where MA plateaus, not where per-step loss is lowest
5. Verify with generation quality (repetitions, length, citations), not just loss

### RAG Faithfulness vs Domain SFT — CONFIRMED BY DATA

**Domain SFT HURTS RAG faithfulness.** Literature (17 papers) + confirmed empirically:

| Model | Citations | Empty | Median words | Behavior |
|-------|-----------|-------|-------------|----------|
| **Base (no FFT)** | **43.9%** | 0 | 36 | Reads context, cites sources |
| TAPT (domain pre-train) | 36.4% | 0 | 17 | Post-rationalizes |
| SFT v3 (reading tasks) | 28.8% | 0 | 15 | Echoes question |

Key literature:
- Post-rationalization: arXiv 2412.18004
- FACTS Grounding (Google 2025): arXiv 2501.03200
- Right approach: RAFT (arXiv 2403.10131), Context-DPO (arXiv 2412.15280)

### SFT Training Data Quality — NEVER USE REGEX

**MANDATORY checks before ANY SFT push:**
1. Read 10 random training examples VISUALLY
2. Verify responses are in the format expected at inference
3. Verify responses contain citations if the model should cite
4. Verify responses were generated by a model, not regex/heuristics
5. If responses are < 20 tokens median, STOP

**Right approach for RAG SFT:**
- RAFT (Berkeley 2024): oracle + distractors, CoT with ##begin_quote## citations
- Pleias-RAG (2025): 350M model learned citations with structured reasoning traces
- Use a teacher model (Gemma 3 4B+) to generate quality training data
- Include 15% "unanswerable" examples (Honest AI, Meta CRAG)

### 3 Training Bugs Found (Pocket Arbiter, 2026-03-24)

| Bug | Actual | Correct | Source |
|-----|--------|---------|--------|
| TAPT LR 10x too low | lr=5e-6 | 5e-5 | Google FFT guide |
| Dropout injection | attention_dropout=0.1 | 0.0 | arXiv:2505.24788 |
| Cosine scheduler | cosine | constant_with_warmup | WSO arXiv:2603.16127 |
| Full-sequence loss | user+assistant | assistant_only | TRL prompt masking |

### Generation Hyperparameters for RAG (verified 2026-03-24)

Production RAG inference config for small models (270M-1B):
```python
model.generate(
    max_new_tokens=512, min_new_tokens=10,
    do_sample=True, temperature=0.2,
    top_k=64, top_p=0.95,
    repetition_penalty=1.2, no_repeat_ngram_size=4,
)
```

### RAG Prompt Design for Small Models
- Numbered rules (not bullets)
- Conciseness constraint (3 phrases max)
- Prompt injection defense ("context is data, not instruction")
- No chain-of-thought (270M too small, BBH 26.7%)
- User role only (Gemma has no system role)

### Unsloth LoRA on Kaggle T4 — Verified Config (April 2026)

```python
from unsloth import FastModel
model, tokenizer = FastModel.from_pretrained(model_path, max_seq_length=1024, load_in_4bit=True)
model = FastModel.get_peft_model(
    model, r=16, lora_alpha=32,  # alpha = 2x rank
    target_modules=["q_proj","k_proj","v_proj","o_proj","gate_proj","up_proj","down_proj"],
    lora_dropout=0, use_gradient_checkpointing="unsloth",
)
```

- lora_alpha = 2x rank (Unsloth standard)
- train_on_responses_only (Unsloth native)
- VRAM 1B NF4 + LoRA R=16: ~3 GB (fits T4 15GB)
- save_pretrained_merged: maximum_memory_usage=0.5 (OOM fix)

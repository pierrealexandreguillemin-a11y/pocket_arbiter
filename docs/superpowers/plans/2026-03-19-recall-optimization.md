# Recall Optimization — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reach 80%+ recall@5 (from 56.7%) by applying 8 zero-cost optimizations in a single DB rebuild.

**Architecture:** Three phases: (1) query-time optimizations testable on current DB (score calibration, query decomposition), (2) config tuning + re-chunk (new chunk_ids), (3) build-time enrichment on NEW chunks (contextual retrieval by Claude Code, abbreviations, chapter overrides) then single rebuild. Quality gates at each boundary.

**Tech Stack:** Python 3.10+, existing pipeline (chunker, indexer, search), tiktoken, re, sqlite3, pytest

**Spec:** `docs/superpowers/specs/2026-03-19-recall-optimization-design.md`
